游戏核心流程 · 详细设计

从案件生成到真相大白，完整游戏循环

🏠 阶段 1 · 首页 & 案件配置

玩家在首页配置本次案件的偏好参数，点击「开始新案件」后触发后端 AI 生成。

可配置参数
  • 案件风格：本格/社会派/刑侦/悬疑
  • 难度：简单（3人2线索）/ 中等（5人4线索）/ 困难（7人6线索）
  • 背景时代：现代/近代/架空
  • 案件类型：谋杀/失踪/盗窃/绑架
  • 额外标签：密室、反转、连续案件...
历史案件存档
  • 已完成的案件卡片展示
  • 评分：S/A/B/C 等级
  • 可回顾完整真相
  • 可重玩（但答案不变）

🎬 阶段 2 · AI 案件生成

后端收到配置后，构造精心设计的 Prompt 调用 Claude API，生成结构化案件数据。

AI 生成的数据结构
案件元信息

标题、类型、难度、简介

人物档案

姓名、年龄、职业、与被害人关系、动机、秘密、对话脚本

线索清单

线索分类（物证/证言/矛盾点）、发现场景、描述文本、关联人物

真相逻辑链

真凶、动机、作案手法、关键证据链、完整时间线、多个误导方向

前端显示氛围动画（打字机效果 + 案件碎片拼合），增强沉浸感。案件数据缓存在服务端 JSON 文件，可重复访问。

📖 阶段 3 · 案件开场

以视觉小说风格呈现案件发现过程，建立氛围和悬念。这是玩家第一次接触案件。

开场要素
  • 旁白叙述：场景描写、气氛渲染
  • 发现者登场：谁发现的、什么反应
  • 案情概要：时间、地点、被害人、初步状况
  • 登场人物一览：嫌疑人列表+简短描述
UI 形式
  • 深色背景 + 场景氛围图（AI 生成或 CSS 绘制）
  • 逐句浮现的文本，手动点击推进
  • 关键信息（时间、地点、被害人）高亮标记
  • 结束后自动进入调查阶段

🔍 阶段 4 · 调查阶段（核心玩法）

玩家在多个场景间切换，与嫌疑人对话、搜寻物证。这是游戏中最自由、时长占比最大的阶段。

场景系统

每个案件 AI 生成 3-6 个调查场景（案发现场、被害人家、嫌疑人工作地、街头、警察局...）。玩家点击场景中的热点进行调查。

🏠 被害人家 🚨 案发现场 🏢 公司办公室 🏪 附近商铺 🏥 法医室 🚔 警察局
对话系统
  • 底部对话面板，角色名 + 立绘/头像
  • 对话逐句显示，点击推进
  • 分支选项：关键节点出现追问/换个话题/施加压力等选项
  • 不同选项可能解锁不同信息或触发人物情绪反应
  • 角色有好感度/信任度隐藏数值，影响信息透露程度
  • 说谎的角色：证词中埋有矛盾，需与其他线索对照
线索收集
  • 场景点击 → 弹出线索发现动画
  • 对话中的关键信息自动标记为证言线索
  • 线索分三类：物证（实物） / 证言（口供） / 矛盾点（逻辑冲突）
  • 右侧始终可见线索栏，实时更新
  • 新线索有闪烁提示，已查看的灰色标记

🧩 阶段 5 · 推理面板

玩家随时可以切换到推理面板（工具栏常驻入口），整理已收集的线索，构建推理。这是从"收集信息"到"形成判断"的关键过渡。

推理画布
  • 所有线索以卡片形式展示在画布上
  • 可拖拽线索卡片自由排列
  • 可连线：在两个线索间画连接线并标注关系（"矛盾" / "佐证" / "时间关联"）
  • 可创建便签：写下自己的推理笔记
  • 可标记嫌疑人：将线索拖到嫌疑人头像上建立关联
人物档案面板
  • 所有嫌疑人/关键人物的卡片
  • 已揭露的动机、不在场证明、矛盾点
  • 人物关系图（自动生成）
  • 嫌疑度用颜色标注（绿→黄→红）

推理面板完全是辅助工具，不强制使用。玩家也可以在脑中推理，直接进入真相揭晓。

⚖️ 阶段 6 · 真相揭晓与评分

提交推理
  • 指认真凶：从嫌疑人列表中选择
  • 陈述动机：从收集的动机线索中选择
  • 还原手法：自由文本输入或从关键线索中选择组合
  • 提交确认：「你确定吗？一旦提交无法修改」
评分与回顾
  • S 级：真凶 + 动机 + 手法全对，发现所有关键线索
  • A 级：真凶正确，动机/手法部分正确
  • B 级：真凶正确但推理过程不够
  • C 级：指认错误
  • 评分后解锁完整真相回顾：时间线动画、真凶动机独白
