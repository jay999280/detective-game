剧本引擎设计 — 如何保证案件精彩

不靠运气，靠结构

🎯 核心思路：分层生成 + 品质校验

不是一次 Prompt 扔给 AI 就完事。分三层生成，每层有品质标准，不合格则重新生成。

第一层：案件骨架 — 核心诡计设计

先不写人物和对话，先设计一个真正有意思的核心诡计。这是精彩案件的地基。

AI 必须完成的诡计设计清单
核心诡计类型 不在场证明诡计 / 身份替换 / 密室 / 叙述性诡计 / 时间诡计 / 凶手即第一发现者...（AI 从诡计库中随机选取并变体）
三层真相结构 表面真相（一开始看起来像什么）→ 调查揭露（收集线索后指向谁）→ 最终真相（反转后的真正答案）
至少一个红鲱鱼 设计 1-2 个精心布置的误导方向：看起来最有动机的人其实无辜、看起来无关的人反而是真凶
关键反转点 案件必须有一个「原来如此」时刻——当玩家发现某条关键线索后，整个案件的面貌翻转
逻辑闭环验证 时间线必须自洽，动机必须合理，手法必须可能。不能有逻辑漏洞。
⚡ 品质闸门：诡计生成后，AI 自我审查——「这个诡计是否有新意？是否至少包含一个读者预料之外的反转？」不通过则重新生成。

第二层：人物血肉 — 每个角色都是嫌疑人

好的推理剧中，每一个嫌疑人都有一个体面的动机，没有人是"纯路人"。

人物设计硬约束
每个人物必须拥有：
  • 一个表面动机（容易被发现的）
  • 一个隐藏秘密（需要深挖才暴露的）
  • 一段与被害人的特殊关系
  • 一个不在场证明（可能是假的）
  • 一句标志性谎言（对话中会出现的）
人物关系网要求：
  • 至少一对秘密关系（不为外人知的）
  • 至少一个角色在保护另一个人
  • 至少一个角色知道的比说的多
  • 人物之间互相矛盾的证词至少 2 处

第三层：对话编织 — 每个字都有信息量

对话不是填充物。每段对话至少承载以下功能之一：提供线索 / 埋下谎言 / 塑造人物 / 制造误导。

对话生成规范
说话风格差异化
  • 每个角色有独特说话方式：语气词、句式长短、专业术语
  • 紧张时、说谎时说话方式有微妙变化
  • 社交地位在对话中体现（恭敬/傲慢/回避）
信息递进节奏
  • 第一轮对话：自报家门 + 表面证词 + 轻量误导
  • 深入追问：暴露矛盾 + 透露秘密 + 情感波动
  • 最后摊牌（关键线索触发）：崩溃/坦白/最后的谎言
谎言设计
  • 真话中掺假：80% 真实 + 20% 谎言才难辨别
  • 每个谎言留下可被推翻的痕迹
  • 有些谎言是善意的（保护他人），有些是恶意的（掩盖罪行）

🔁 最终品质校验

全案生成后，AI 以「挑剔玩家」的视角检查质量，不合格项触发针对性重新生成。

不通过标准
  • 真凶一眼就能看出来（前 3 条线索就暴露）
  • 存在逻辑漏洞（时间线对不上）
  • 人物动机不合理
  • 关键线索与真相无关
  • 反转过于牵强
  • 有角色毫无存在意义
通过标准
  • 核心诡计有原创性
  • 3 个嫌疑人中至少有 2 个看起来像真凶
  • 反转点能带来真正的「原来如此」
  • 每条线索都有存在的意义
  • 对话读起来自然不僵硬
  • 真相在回看时感觉「合理且巧妙」

📐 Prompt 架构

不是一条 Prompt，而是编排好的生成流水线：

1. System Prompt：设定 AI 为资深推理小说作家角色
2. 参数注入：{风格} {难度} {案件类型} {角色数}
3. 第一轮 → 生成诡计骨架 + 逻辑自检
4. 第二轮 → 基于通过校验的诡计，生成人物 + 关系网
5. 第三轮 → 基于诡计+人物，生成对话脚本 + 线索分布
6. 最终轮 → 品质自审 + 输出结构化 JSON
